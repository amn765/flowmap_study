# 阶段 4 · MeanFlow 核心：平均速度与恒等式（约 4-5 天）

> 目标：这是整个网站的心脏。亲手推一遍 **MeanFlow 恒等式**，理解「用平均速度替代瞬时速度」为什么能让一步生成成立，并写出采样公式。推导只用到微积分基本定理和链式法则 —— 关键是把每一步的物理含义讲清楚。
> （梯度提示：阶段 1 你已经会「积分 ↔ 求导」，阶段 3 你已见过「区间当条件」。这里只是把它们合成一个干净的恒等式。）

主线资料：⭐ *Mean Flows for One-step Generative Modeling*（Geng, Deng, Bai, Kolter, He, 2025）第 3 节。建议打印出来逐行推。

约定：用线性插值路径 `z_t = (1-t) x + t ε`，t∈[0,1]，`z_0 = x`（数据），`z_1 = ε`（噪声）。瞬时速度（条件流匹配目标）为 `v(z_t, t) = ε - x`。

## 任务清单

### ☐ 4-1 平均速度 average velocity 的定义
- 定义从 r 到 t 的**平均速度**：把这段位移除以时间区间长度。
  ```
  u(z, r, t) := 1/(t - r) * ∫_r^t v(z_τ, τ) dτ
  ```
- 等价地，乘开就是**位移 = 平均速度 × 区间**：
  ```
  (t - r) * u(z_t, r, t) = ∫_r^t v dτ = z_t - z_r
  ```
- 直觉：瞬时速度 `v` 是「此刻多快」；平均速度 `u` 是「这段路平均多快」。学会 u，就能一步跨过整段。
- 思考题：当 `r → t` 时，平均速度 `u(z, r, t)` 趋于什么？（这是后面边界条件的关键。）
- **完成标志**：能默写 u 的定义，并解释「(t-r)·u = 位移」这条等式。

### ☐ 4-2 ⭐ MeanFlow 恒等式推导
- 起点：`(t - r) · u(z_t, r, t) = ∫_r^t v(z_τ, τ) dτ`。
- 固定 r，把两边对 **t** 求**全导数** `d/dt`（注意 `z_t` 也依赖 t）。
- 右边：由微积分基本定理（对积分上限求导）：
  ```
  d/dt ∫_r^t v dτ = v(z_t, t)
  ```
- 左边：乘积求导，
  ```
  d/dt [ (t-r) u ] = u(z_t, r, t) + (t - r) * du/dt
  ```
- 令两边相等，整理得 **MeanFlow 恒等式**：
  ```
  u(z_t, r, t) = v(z_t, t) - (t - r) * du/dt
  ```
- 它把「我们想学的平均速度 u」用「已知的瞬时速度 v（= ε - x）」和「u 自己的全导数」表达出来 —— 这就是训练监督信号的来源。
- **完成标志**：能在白纸上独立从定义推到 `u = v - (t-r) du/dt`，每一步都说得出依据。

### ☐ 4-3 全导数 d/dt 的展开
- 这里的 `du/dt` 是**沿轨迹的全导数**（total derivative），把 u 的三个自变量都算上。因为 r 固定、t 是自变量、`z_t` 随 t 变：
  ```
  du/dt = ∂u/∂z · dz_t/dt + ∂u/∂r · dr/dt + ∂u/∂t
        = ∂u/∂z · v(z_t, t) + 0 + ∂u/∂t
        = v · ∂_z u + ∂_t u
  ```
  （`dr/dt = 0` 因为 r 固定；`dz_t/dt = v(z_t,t)` 沿真实轨迹。）
- 所以恒等式可写成只含偏导的形式：
  ```
  u(z_t, r, t) = v - (t - r) * ( v · ∂_z u + ∂_t u )
  ```
- 思考题：`v · ∂_z u + ∂_t u` 正好是函数 `u` 沿方向 `(v, 0, 1)`（对应 z, r, t 三个输入）的方向导数。这个观察会在阶段 5 直接对应一行 JVP 代码。
- **完成标志**：能写出 `du/dt = v·∂_z u + ∂_t u`，并解释每一项及为何 `∂_r` 项消失。

### ☐ 4-4 网络参数化与边界 r = t
- 用网络 `u_θ(z, r, t)` 拟合平均速度。训练目标由恒等式给出（stop-gradient 见阶段 5）：
  ```
  u_tgt = v - (t - r) * ( v · ∂_z u_θ + ∂_t u_θ )
  L = || u_θ(z_t, r, t) - sg(u_tgt) ||²
  ```
- **边界情形 r = t**：区间长度为 0，恒等式退化为 `u(z_t, t, t) = v(z_t, t)` —— 即平均速度在零区间上等于瞬时速度。这恰好是普通 flow matching 损失！所以 MeanFlow 在 `r=t` 的样本上就是在做标准 FM，把「会算瞬时速度」作为地基。
- 思考题：如果训练时只采 `r=t` 会怎样？只采 `r≠t` 又会怎样？为什么两者都要、还要按比例混？
- **完成标志**：能写出训练目标，并解释 `r=t` 退化为 flow matching 的意义。

### ☐ 4-5 一步 / 少步采样公式
- 采样直接用「位移 = 平均速度 × 区间」，无需积分：
  ```
  z_r = z_t - (t - r) * u_θ(z_t, r, t)
  ```
- **一步生成**：从噪声 `z_1 = ε`，取 t=1, r=0，一次前向：
  ```
  x ≈ z_0 = z_1 - (1 - 0) * u_θ(z_1, 0, 1) = ε - u_θ(ε, 0, 1)
  ```
  NFE = 1。
- **少步生成**：把 [0,1] 分成几段，逐段套同一公式（如 2~4 步），质量更高、仍远少于扩散。
- 思考题：为什么用平均速度做一步采样是**精确**的（只要 u 学对），而用瞬时速度做一步是**有数值误差**的？
- **完成标志**：能写出采样递推，并说明一步采样为何理论上无离散化误差。
